# Book templates

A *book template* describes how a fresh book repository is initialized.
Today the only template is `memoir-cowork`, extracted from a real
working memoir-in-progress repo. Future templates (essay, novel,
journalistic) follow the same shape.

## Layout

```
templates/{template_id}/
├── manifest.json
├── CLAUDE.md.tmpl
├── outline.md.tmpl
├── README.md.tmpl
├── acerca-de-mi.md.tmpl
├── nota-del-autor.md.tmpl
├── notas/
│   ├── seguimiento.md.tmpl
│   ├── inconsistencias.md.tmpl
│   ├── permisos.md.tmpl
│   ├── cronologia.md.tmpl
│   └── como-retomar.md.tmpl
└── entrevistas/
    ├── _plantilla-introduccion.md.tmpl
    └── _plantilla-guia-tercero.md.tmpl
```

Each `.tmpl` file is plain markdown (or JSON) with `{{PLACEHOLDER}}`
tokens that the scaffolder substitutes when applying the template.

## `manifest.json` shape

```jsonc
{
  "id": "memoir-cowork",
  "name": "Memoir in Cowork",
  "description": "Structure for a memoir worked in Cowork-with-Claude mode",
  "language": "es",
  "placeholders": [
    { "key": "TITULO",        "label": "Book title",                "required": true,  "example": "..." },
    { "key": "SUBTITULO",     "label": "Subtitle (optional)",       "required": false, "example": "..." },
    { "key": "TESIS",         "label": "Central thesis or question",  "required": true,  "example": "..." },
    { "key": "LECTOR_IDEAL",  "label": "Ideal reader, in one sentence","required": true,  "example": "..." },
    { "key": "TONO",          "label": "Tone (e.g. intimate, ironic)","required": true,  "example": "..." },
    { "key": "AUTOR_NOMBRE",  "label": "Author name",                 "required": true },
    { "key": "TEMA_PRINCIPAL","label": "Main theme in 2-4 words",     "required": true,  "example": "..." },
    { "key": "ANIO_INICIO",   "label": "Year the story begins",       "required": false }
  ],
  "directories": [
    "capitulos",
    "entrevistas/autor",
    "entrevistas/terceros",
    "investigacion",
    "notas"
  ],
  "files": [
    { "src": "CLAUDE.md.tmpl",                            "dst": "CLAUDE.md" },
    { "src": "outline.md.tmpl",                           "dst": "outline.md" },
    { "src": "acerca-de-mi.md.tmpl",                      "dst": "acerca-de-mi.md" },
    { "src": "nota-del-autor.md.tmpl",                    "dst": "nota-del-autor.md" },
    { "src": "notas/seguimiento.md.tmpl",                 "dst": "notas/seguimiento.md" },
    { "src": "notas/inconsistencias.md.tmpl",             "dst": "notas/inconsistencias.md" },
    { "src": "notas/permisos.md.tmpl",                    "dst": "notas/permisos.md" },
    { "src": "notas/cronologia.md.tmpl",                  "dst": "notas/cronologia.md" },
    { "src": "entrevistas/_plantilla-introduccion.md.tmpl","dst": "entrevistas/terceros/_plantilla-introduccion.md" },
    { "src": "entrevistas/_plantilla-guia-tercero.md.tmpl","dst": "entrevistas/terceros/_plantilla-guia.md" },
    { "src": "README.md.tmpl",                            "dst": "README.md" }
  ],
  "enabled_modes": ["interviewer", "architect", "writer", "researcher"],
  "first_commit_message": "Initial scaffold from memoir-cowork template"
}
```

## Where templates come from

The first one (`memoir-cowork`) was generated by *reading* an existing
memoir-in-progress book repo and extracting its structure: which folders
exist, which files, which section headings, which conventions (no YAML
frontmatter, kebab-case filenames, `<!-- FUENTES: -->` comments at
chapter ends, `_pendiente-` prefix for unfinished material).

The book draft itself was not modified. Only the *shape* is in
`templates/memoir-cowork/`.

The placeholder keys stay in Spanish because the template targets
Spanish-language memoirs. An English-language template would either
duplicate keys (`TITLE`, `THESIS`, …) or use the same Spanish keys with
English placeholder content — open question, see
[`docs/open-decisions.md`](open-decisions.md).

## Validation

`pnpm validate:templates` ([scripts/validate-templates.ts](../scripts/validate-templates.ts))
checks every template directory:

- `manifest.json` parses.
- Every `files[i].src` exists on disk.
- Every `{{PLACEHOLDER}}` referenced in any `.tmpl` is declared in
  `manifest.json:placeholders` (modulo a small allowlist of structural
  helpers like `{{TITULO_KEBAB}}`).

## Applying a template (planned)

`lib/repo/scaffolder.ts` will accept
`(template_id, placeholders, target_repo_path)`, copy each `files[i]`
substituting placeholders, create the listed `directories`, and make
the first commit with `manifest.json:first_commit_message`.

The "create new book" UI hooking this up is in
[`docs/open-decisions.md`](open-decisions.md). Today the author
*connects* an existing repo; scaffolding from a template is the next
big UI piece.

## Adding a new template

1. Create a directory under `templates/<your-id>/`.
2. Write `manifest.json` declaring placeholders, files, directories,
   and `enabled_modes`.
3. Add `.tmpl` files. Use `{{PLACEHOLDER}}` tokens for substitution and
   `{{#KEY}}…{{/KEY}}` for optional sections.
4. `pnpm validate:templates` until it's green.
5. Open a PR. Notes on what kind of book the template targets, and a
   pointer to a real repo it was modelled after if any, are very
   welcome.
